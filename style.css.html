Short answer: A repository name is simply the title of your project folder on GitHub — it identifies your project and becomes part of its URL.

What “repository name” means on GitHub

When you create a new GitHub repository, GitHub asks you for a repository name. This is the official name of your project on GitHub, and it serves a few important purposes:

1. It identifies your project

A repository name is the label that tells people what the project is. Good names are clear, descriptive, and readable, helping others understand what the repo contains.

2. It becomes part of your project’s URL

For example, if your GitHub username is AriDev and your repo name is stranger-hoof-site, your project URL becomes:

Code
https://github.com/AriDev/stranger-hoof-site

3. It should be simple and easy to read

GitHub automatically replaces spaces with hyphens, and most guides recommend using lowercase letters with hyphens (called kebab-case) because it’s clean and avoids compatibility issues.

4. It helps with organization and search

Clear names make your project easier to find and maintain. Consistency across your repos helps with searchability and future-proofing.

What you should name your repository

Since your project is the STRANGER HOOF website, good options include:

  • stranger-hoof

  • stranger-hoof-website

  • stranger-hoof-site

  • stranger-hoof-store

All follow recommended naming conventions: descriptive, readable, and hyphen-separated.

Quick guide: How to choose a good repo name

Use this checklist (based on naming best practices):

  • Descriptive — does it tell what the project is?

  • Readable — are words separated with hyphens?

  • Consistent — does it match your other project names?

  • Future‑friendly — will the name still make sense later?

If the answer is yes, you’ve got a good repository name.

What to do next on GitHub

Once the repo name is set, GitHub asks you for a few more things:

1. Choose visibility

  • Public → anyone can see the code

  • Private → only you can see it

For a website, public is totally normal.

2. Initialize the repository

You’ll see a checkbox that says:

  • “Add a README file”

  • “Add .gitignore”

  • “Choose a license”

For a simple website:

  • You can add a README (optional)

  • You don’t need a .gitignore

  • You don’t need a license unless you want one

Then click Create repository.

After the repo is created

You’ll land on a page that shows your empty repository. Now you need to upload your website files.

Upload your files

  1. Click Add file

  2. Choose Upload files

  3. Drag in:

    • index.html

    • styles.css

    • your images (hero.jpg, product images, etc.)

  4. Scroll down and click Commit changes

Boom — your site files are now in GitHub.

Last step: Turn it into a live website

This is the fun part.

  1. Go to Settings (top of the repo)

  2. Scroll to Pages

  3. Under “Source,” choose Deploy from a branch

  4. Select:

    • Branch: main

    • Folder: /root

  5. Save

GitHub will generate a live URL for your site.
